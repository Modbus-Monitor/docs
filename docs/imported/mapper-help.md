## Introduction

<div class="wp-block-columns is-layout-flex wp-container-core-columns-is-layout-28f84493 wp-block-columns-is-layout-flex">

<div class="wp-block-column is-layout-flow wp-block-column-is-layout-flow">

Welcome to the **Modbus Mapper Pro User’s Manual**, your guide to
mastering this innovative application for simplifying Modbus RTU
communication. Designed to help you navigate the software’s powerful
features, this manual provides step-by-step instructions and tips for
installation, configuration, and efficient usage.

**Modbus Mapper Pro** is an innovative application designed to address
and simplify the most complex Modbus RTU communication challenges. The
application is built with two core purposes that set it apart in the
field of industrial automation tools:

1.  **Dynamic Modbus Mapping**: Modbus Mapper Pro dynamically constructs
    a detailed Modbus map in real-time as it monitors communications.
    This map decodes and translates raw Modbus frames into
    human-readable values such as strings, integers, and floats,
    enabling users to view the exchanged data in a natural,
    comprehensible format. The results are displayed in an intuitive
    Excel-style grid, providing a clear and actionable visual
    representation of device data and register activity. This feature
    eliminates the need for manual decoding of Modbus frames, making
    diagnostics, system optimization, and troubleshooting a breeze.
2.  **Sharing a Single Slave Port with Multiple Masters**: Modbus Mapper
    Pro enables the seamless sharing of a single Modbus RTU server
    (Slave) with multiple Clients (Masters). This functionality ensures
    efficient and conflict-free communication, even in complex
    multi-client setups. Additionally, the tool allows silent monitoring
    of RS485 RTU devices without disrupting their operation, making it
    an invaluable asset for engineers and technicians working with live
    systems.

By combining real-time data monitoring with these unique capabilities,
**Modbus Mapper Pro** empowers users with a unique way to monitor the
live data exchanges between the Modbus RTU client and server. Its

### Connecting to Devices

Modbus Mapper Pro uses standard serial ports available in Windows to
communicate with devices or to sniff data on existing RTU networks. The
available serial ports can be found under the **“Ports (COM & LPT)”**
section in the Windows Device Manager.

#### Tips for Optimal Serial Port Performance

1.  **Choose Quality Hardware**:  
    For reliable communication, use high-quality serial port adapters
    with low-latency hardware and robust drivers. These ensure stability
    across various Windows states, such as Sleep or Hibernate, without
    locking up or losing functionality.
2.  **RS485 Sniffing Considerations**:  
    When monitoring existing traffic on an RS485 network (Mode 2),
    select a reliable RS485 adapter that can silently listen to the
    communication without interfering. It is crucial to use adapters
    capable of starting and staying in **Receive Mode** to avoid
    interrupting the existing communication flow.
3.  **Recommended Serial Port Manufacturers**:  
    Modbus Mapper Pro works seamlessly with a variety of USB-to-Serial
    adapters, and the following brands have been thoroughly tested for
    performance and reliability:
    - **FTDI**: Known for its high-quality chipsets and drivers that
      provide consistent performance.
    - **Moxa**: A trusted name in industrial automation, offering
      durable and efficient adapters suitable for demanding
      environments.
    - **Silicon Labs (CP210x)**: Provides affordable and reliable
      USB-to-Serial solutions with well-supported drivers.
    - **Prolific Technology (PL2303)**: A widely-used option that works
      well in less demanding environments. Ensure you use genuine
      products to avoid compatibility issues.
    - **StarTech**: Offers industrial-grade USB-to-RS485/RS232
      converters, ideal for professionals.
4.  **Additional Considerations**:
    - Always verify the driver version of your adapter to ensure
      compatibility with the latest Windows updates.
    - Avoid counterfeit adapters, as they often have unstable drivers
      that can lead to communication failures or device lock-ups.

By adhering to these guidelines and using trusted hardware, you can
ensure stable and reliable Modbus communication for both data exchange
and monitoring tasks.

**Two Modes of Operation**

**Modbus Mapper Pro** supports two distinct modes operation for
versatile functionality.

1.  **Mode 1 – Pass-Through or Multiplexer Mode**:  
    In this mode, data from a serial port connected to Client 1 or 2 is
    passed through internal memory to the Server port. This setup allows
    you to monitor and analyze the traffic while facilitating
    communication between clients and servers. The built-in
    **multiplexer** ensures seamless operation by automatically
    prioritizing requests from both clients. It synchronizes these
    requests, managing their timing and sequencing to prevent conflicts
    before forwarding them to the server port. This intelligent
    synchronization ensures efficient, conflict-free communication,
    making it possible to analyze data in real time without disrupting
    the workflow.
2.  **Mode 2 – Listen-Only Mode**:  
    This mode is ideal for monitoring existing communication on an RTU
    network without disrupting ongoing operations. It offers a
    non-intrusive way to capture and analyze data exchanged between a
    Client and a Server. To enable this mode, ensure that a single
    serial port is connected and that the **‘Listen Only’** switch is
    turned on. This configuration ensures passive monitoring while
    maintaining the integrity of the network’s communication.
